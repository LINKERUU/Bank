# Binance - Glossary of Key Terms

## Account (User Account)
A unique identity within the Binance system, created during registration. It is associated with a user's personal data, list of accounts, cards, and transaction history.

## Authentication
The process of verifying a user's identity, typically by validating e-mail and password. Successful authentication results in the correct login and password.

## Authorization
The process of determining and enforcing what actions an authenticated user is permitted to perform (e.g., viewing accounts, transactions, cards, and statistics).

## Account
A financial account held by a user, including balances and associated cards.

## Card
A payment card linked to a user's account, which can be viewed and analyzed for transactions.

## Transaction
A record of a financial operation, including date, amount, type (income/expense), and category.

## Controller
A component providing the REST API interface, handling incoming client requests, invoking business logic, and returning responses.

## DTO (Data Transfer Object) / Mapper
A pattern used to transfer data between software subsystems. DTOs carry data between processes, while Mappers handle conversion between entities and DTOs.

## Service
A component encapsulating the core business logic, operating on accounts, cards, and transactions to perform specific operations.

## Statistics
Aggregated and analyzed data providing insights into user spending, income, and card usage.

## User
A registered individual interacting with the Binance  system. Users can assume roles such as standard user or administrator, which determine their permissions.

## Notification Service
A component responsible for sending transactional notifications, such as password reset instructions or account alerts, via e-mail.
