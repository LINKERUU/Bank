# 1 Введение
<a name="appointment"/>

## 1.1 Назначение
Документ определяет функциональные и нефункциональные требования к веб-приложению **Binance**, предназначенному для просмотра и анализа финансовых данных пользователей, включая счета, карты, транзакции и статистику. Документ служит руководством для команды разработки и тестирования.

<a name="business_requirements"/>

## 1.2 Бизнес-требования
<a name="initial_data"/>

### 1.2.1 Исходные данные
Пользователи нуждаются в простом способе контроля своих финансовых данных через интернет. Существующие банковские приложения часто перегружены функциями, усложняющими использование.

<a name="business_opportunities"/>

### 1.2.2 Возможности бизнеса
Приложение позволит:
- Предоставить пользователям быстрый и безопасный доступ к их финансовой информации.
- Отслеживать движение средств и анализировать расходы.
- Предоставлять учебную платформу для базовых банковских операций.
- Поддерживать дальнейшее расширение функционала, например добавление кредитования и инвестиционных инструментов.

<a name="project_boundary"/>

### 1.2.3 Границы проекта
**Bank App** позволяет зарегистрированным пользователям:
- Входить в учетную запись и восстанавливать пароль.
- Просматривать счета и баланс.
- Просматривать карты и их данные.
- Просматривать историю транзакций.
- Анализировать статистику расходов и поступлений.

Анонимные пользователи могут:
- Просматривать форму авторизации и регистрации.
- Восстанавливать пароль через логин.

<a name="analogues"/>

## 1.3 Аналоги
СберБанк Онлайн, Тинькофф, Альфа-Банк.  
Преимущества **Binance**: упрощённый интерфейс для учебных целей, наглядное управление финансовыми данными, открытый код.

### 3.1.2 Ограничения и исключения
1. Доступ только после авторизации.

<a name="non-functional_requirements"/>

## 3.2 Нефункциональные требования
<a name="quality_attributes"/>

### 3.2.1 Атрибуты качества
- **Надежность:** uptime ≥ 99%
- **Безопасность:**  хэширование паролей, защита от SQL-инъекций, CSRF, XSS
- **Производительность:** отклик ≤ 1 сек
- **Масштабируемость:** поддержка увеличения числа пользователей
- **Удобство использования:** до 3 кликов к ключевым данным
- **Поддерживаемость:** модульная структура, тесты ≥ 80% покрытия

### 3.2.2 Внешние интерфейсы
- REST API
- Swagger/OpenAPI

### 3.2.3 Ограничения
- Работа только в веб-браузере
- Поддержка настольных и мобильных устройств
